1 SRS060

Working State:

1 SRS061

Software interrupt BatLow shall trigger in GGVTS when battery of GGVTs is lower than Ggvtsbatlvltostartggvts.

1 SRS062

As soon as BatLow iterrupt occurs. A wake sequence shall be sent to BMS from GGVTS.

1.1 SRS002 SRS002

As soon as GGVTS receives request to turn ON Park Mode, GGVTS shall perform Park Mode functionality.

1.2 SRS003 SRS003

As soon as button to turn Park Mode is pressed, GGVTS shall perform Park Mode functionality.

1.3 SRS004 SRS004

As soon as Park Mode functionality started, GGVTS shall request for battery statuses of host as well as GGVTS from charging Module (CM) and wait for batStatusWaitMax for the status.

1.4 SRS005 SRS005

If battery statuses received in batStatusWaitMax, GGVTS shall check if battery level of GGVTS is equal to or above GGVTSbatLvltoStartGGVTS.

1.5 SRS126 SRS126

If batter level for GGVTS is more than or equal to GGVTSbatLvltoStartGGVTS, GGVTS shall check if host battery level is more than hostbatLvltoStartGGVTS.

1.6 SRS007 SRS007

If battery level of GGVTS is less than GGVTSbatLvltoStartGGVTS and host battery level is above hostbatlvltoStartGGVTSchrg GGVTS shall request CM to enable charging for GGVTS battery until GGVTS battery reaches for GGVTSMaxBatteryLvl.

1.7 SRS008 SRS008

As soon as Park Mode functionality started, GGVTS shall check GPS status.

1.8 SRS009 SRS009

If GPS is working fine, GGVTS shall perform location status check.

1.9 SRS010 SRS010

If location is fixed and GPS is working fine, GGVTS shall check for GSM functionality by checking range level of GSM.

1.10 SRS011 SRS011

If GSM have enough range level, GGVTS shall fetch current location and store it.

1.11 SRS012 SRS012

As soon as current location is stored, GGVTS shall compare current location with stored location (stored in previous requirement) every 10 mins.

1.12 SRS013 SRS013

If anything goes wrong, GGVTS shall behave as per Host Crammed Situation.

2 Host Crammed Situation SRS014

Host Crammed Situation

2.1 SRS015 SRS015

During active Park Mode, If current location of host is deviate more than maxDevaiationallowedinlocation from the stored location then a communication with owner shall start via SMS to inform user about theft in every 30 seconds until user responds and this situation shall be known as Theft Alert.

2.2 SRS016 SRS016

In case of Theft Alert, If GGVTS receives request to stop engine from User, GGVTS shall disable all the power of engine and take control of power and host shall be turned off instantly and user should be informed about location of host stopped and this situation will be known as Host Crammed.

2.3 SRS017 SRS017

In case of Host Crammed situation, GGVTS shall not have any impact from Park Mode functionality button active/Inactive.

2.4 SRS018 SRS018

During Host Crammed Situation, If host is tried to turn on again either manually or by sending request, GGVTS shall ignore all the requests and shall not provide power to engine to turn on.

2.5 SRS019 SRS019

In case of security violation, If GGVTS receives request to turn OFF alerts from user, GGVTS shall ask user for security overwrite PIN.

2.6 SRS020 SRS020

Once user confirms the PIN, GGVTS shall behave as Security Overridden.

2.7 SRS021 SRS021

In case of Pin Mismatch, GGVTS should inform Soldiers.

2.8 SRS022 SRS022

In case of Fake Host Crammed situation, GGVTS can be put on POWER OFF mode with power off request from owner or via power off switch.

3.1 Location Functionality of GGVTS SRS024

Location Functionality of GGVTS

3.1.1 SRS025 SRS025

Upon power ON of GGVTS, GPS shall be initialized by following GPSInitFunctionality.

3.1.2 SRS026 SRS026

As soon as GPS is initialized, GGVTS shall try to fix the location as 2D or 3D as per LocationFixingFunctionality.

3.1.3 SRS027 SRS027

If Location is fixed, User shall be informed as GPSWorkingFineMessage.

3.1.4 SRS028 SRS028

If location is not fixed as per LocationFixingFunctionality, GPS working flag shall be turn OFF and user shall be informed as a GPSNotWorkingMessage.

3.1.5 SRS029 SRS029

If GPS working flag is OFF and a GPS related functionality is requested by user, user shall be informed as a GPSNotWorkingMessage.

3.1.6 SRS030 SRS030

If Location fixed as per LocationFixingFunctionality, GPS working flag shall be turn ON and GGVTS shall perform further functionalities.

3.1.7 SRS031 SRS031

Upon receiving location related request from user, GGVTS shall check for the GPS working flag status.

3.1.8 SRS032 SRS032

Once GPS working flag is checked as ON and location is fixed with 2D or 3D, GGVTS shall reply with current location of host.

3.1.9 SRS033 SRS033

Upon receiving continuous location request, GGVTS shall continuously send location to the user Location as per ContinuousLocationFunctionality.

4 Engine Power Functionality of GGVTS SRS034

Engine Power Functionality of GGVTS

4.1 SRS035 SRS035

Upon receiving Power On request from user, GGVTS shall check if power of host is already ON. If not GGVTS shall Power On host, else ignore the message by following ignore message light pattern.

4.2 SRS036 SRS036

Upon receiving Power Off request from user, GGVTS shall check if power of host is already OFF. If not GGVTS shall Power Off host, else ignore the message by following ignore message light pattern.

5 Message Handler SRS127

Message Handler

5.1 SRS128 SRS128

As soon as GGVTS receives a new message, sender number and message content shall be extracted and check for below conditions.

5.1.1 SRS129 SRS129

Cond 1: Sender number and stored user number are same.

5.1.2 SRS130 SRS130

Cond 2: Extracted message is from list of functionalities.

5.2 SRS131 SRS131

If Condition1 and Condition2 are true, GGVTS shall perform requested functionality. Refer SRS128

5.3 SRS132 SRS132

If Condition1 is true and Condition2 is false, user shall be informed as a InvalidOption. Refer SRS128

5.4 SRS133 SRS133

If Condition1 is false and Condition2 is true, user should be informed as a SecurityViolationMessage. Refer SRS128

5.5 SRS134 SRS134

If Condition1 and Condition2 are false, user should be informed as a InformationMessage. Refer Section “Various Message Formats”
